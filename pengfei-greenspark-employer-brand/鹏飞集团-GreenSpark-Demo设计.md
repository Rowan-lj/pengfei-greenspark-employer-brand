

## 一、Demo 定位

**一句话描述**：一个飞书智能机器人，学生或HR输入"大学名称 + 专业名称"，自动生成一份定制化的校招宣讲文案——包含个人化邀请语、专业对口的鹏飞业务介绍、职业成长路径展望，以及适合该专业学生转发的小红书/B站内容草稿。

**演示场景**：
- **场景 A**：HR 在飞书群里准备 30 所高校的差异化物料，@机器人 批量生成
- **场景 B**：学生在招聘会现场扫码添加机器人，输入自己的专业，即刻看到"如果我去鹏飞能做什么"

**设计溯源**

> 以下表格展示 GreenSpark Demo 的设计决策如何直接源于竞品调研和信号类型学分析。这不是"灵光一现"的设计，而是"有据可查的推导"。

| 竞品分析发现                                                    | 对应Demo设计决策                                                                                                                   | 如果不这样做会怎样                                                    |
| --------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------ |
| **玛氏：AI 内容管线已验证可行**                                       | Demo 底层用 LLM API（而非硬编码模板）驱动内容生成——因为只有 AI 能做到"输入 985 化工硕 → 生成一套叙事；输入双非计算机本 → 生成另一套叙事"                                         | 回到"一套物料发全校"——产能锁死精准触达                                        |
| **玛氏小红书：账号人格化的"打工人视角"（Type 3 平视叙事）**                      | 机器人回复语气设计为**朋友聊天而非客服应答**——"你好！告诉我你的专业，我帮你看你在鹏飞能做什么 😎" → 而不是"您好，我可以帮您了解鹏飞集团的岗位信息"                                            | 学生感觉在和"一个系统"对话，而非"一个懂我的人"——信任度从 Type 3 掉到 Type 4             |
| **蔚来/特斯拉：员工自发内容（Type 1 有机体验）传播力是官方内容的 10×**               | Demo 输出物中包含**校园大使一键转发素材包**（小红书帖含标题/正文/配图建议/话题标签）——大使不是"自己想了再写"，是"拿到就能发"                                                      | 大使还是只能转推文——依然是 Type 4 信号                                     |
| **字节跳动："负面标签不劝退"的完整信号包效应**                                | EVP 信号转译引擎**不回避鹏飞的"短板"**——对"去山西是不是很无聊"不防御不否认，而是翻转为"太原没有三里屯，但你的薪资有 3 倍购买力"                                                    | 防御式回应 → 学生直觉判断"他们在掩饰"→ 所有正面信号的可信度打折                          |
| **宁德时代/比亚迪："按校分级薪资体系"创造了"规则透明"的信号**                       | Demo 输出的业务介绍中**每个数据都可溯源**——"2 万吨/年制氢基地已投产"不是形容词，是可验证的事实                                                                      | 内容变成"吹牛"——学生分不清哪些是真的哪些是编的，默认全不信                              |
| **猎聘/实习僧等传统校招工具：只做"流程"，不做"翻译"**                           | Demo 的核心交互不是"岗位搜索"（他们已经有这个），而是**"你的专业→在鹏飞能干什么"的认知翻译**                                                                        | 又做成了一个自动回复版岗位 JD 查询器——跟 100 个校招小程序没区别                        |
| **生态系统理论：个体行为改变需要环境支持**                                   | Demo 的输出物中有**"推荐分发渠道 + 最佳发布时间"**策略建议——不是只给素材，而是告诉使用者"在哪儿发、什么时候发、怎么发"                                                         | 素材好但分发策略错误 → 好内容被埋没 → 数据差 → 方案被判"无效"                         |
| **玛氏 AI 内容管线的"AI→人工调优"工作流**                               | Demo 设计为**AI 生成初稿 + 人工调优确认**的双阶段工作流（而非 AI 直接发布）——机器人回复底部有"换个语气""再专业一点""加个冷笑话"的二次调整按钮                                         | 内容千篇一律 → 学生刷到第三篇同一风格的帖子就会"审美疲劳"                              |
| **求职者流程体验是雇主品牌最强的正面/负面信号**（好想你大枣礼包→自发传播；HR 48h 无反馈→小红书吐槽） | Demo 不只是"内容生成工具"——它还承载**流程体验触点**：① 机器人 48h 自动催办未回复的投递进度；② 宣讲会现场扫描二维码 → 实时输出"你的专业在鹏飞能做什么"，让"来了就有收获"变成可传播的记忆点                  | 内容做得好，但流程体验差 → 学生在小红书吐槽"鹏飞校招投了没反应"→ AI 生成再多的好内容也补不回来         |
| **60%+ 应届生求职方向迷茫**——不知道自己适合什么岗位（摘自 2025 届校招调研）            | Demo 的第一步不是"输出鹏飞介绍"——而是**"先帮你搞清楚你适合什么方向"**的轻量职业诊断：5 个行业方向选择题 + 专业适配分析 → 让学生从"我什么都看不懂"到"我知道至少有两个方向适合我" → 然后再基于方向偏好输出定制化鹏飞业务介绍 | 学生不知道自己适合什么 → 你告诉 TA "鹏飞氢能很好" → TA 的反应是"哦，然后呢？我适合吗？"——信息接收失败 |

### 2.3 求职方向诊断——Demo 的前置交互流

> **设计依据**：2025 届校招调研显示超过 60% 的应届生"不太清楚自己适合什么岗位"。学生在做出"我要不要投鹏飞"之前，需要先回答一个更根本的问题："我适合做什么？" 如果 GreenSpark 能帮助一个迷茫的学生完成这个判断——即使最终他没有投鹏飞——鹏飞在他心中的品牌印象已经被"这家公司首先想帮我，而不是首先想招我"锚定了。

**这个交互设计的三个关键原则**：

1. **不先出答案，先帮你问对问题**。传统校招工具的逻辑是"搜索"——学生知道自己想做什么，然后搜。GreenSpark 的逻辑是"探索"——学生不知道自己适合什么，机器人帮他缩小范围。这直接回应了 60%+ 应届生求职方向迷茫的现实。

2. **即使你不选鹏飞，这次对话也是有价值的**。如果学生选了"汽车及零部件"，机器人不会强行说"但我们也有氢能重卡啊你可以考虑一下"——而是真诚地给汽车行业的中立分析和投递建议，然后说"如果以后想看新能源装备方向，随时回来找我"。这遵循了社会工作的"案主自决"原则——**尊重学生的自主选择，远比转化一个投递更重要**。而且，一个被真诚对待的学生，未来有 3-5 年校招窗口期可能重新考虑鹏飞。

3. **"鹏飞在这里"是一个轻推（nudge），不是一次推销（pitch）**。选项旁边的"← 鹏飞在这里！"只是一个微小的视觉提示——不干扰选择，但建立了初步的认知关联。行为经济学中的"锚定效应"（Tversky & Kahneman, 1974）表明，这种微弱的初步曝光在后续信息接收中会产生显著的认知便利性——"哦，我之前听过这家公司"。

## 三、核心技术设计

### 3.1 专业→业务线映射模型

这是整个 Demo 最核心的"智能"所在——不是简单的一对一映射，而是**多对多的加权匹配矩阵**。

```
                        鹏飞业务线
                        ──────────
              氢能全产业链  零碳物流  氢能装备  5G数智化  碳交易
              ──────────  ──────  ──────  ──────  ────
化学工程         0.95        0.3     0.4      0.1     0.2
材料科学         0.85        0.1     0.7      0.1     0.1
机械工程         0.3         0.3     0.9      0.2     0.1
计算机科学       0.1         0.6     0.1      0.95    0.1
环境工程         0.6         0.7     0.2      0.2     0.8
电气工程         0.7         0.3     0.5      0.3     0.2
车辆工程         0.4         0.8     0.6      0.3     0.1
经管/经济学       0.1         0.3     0.1      0.1     0.9
物流管理         0.1         0.95    0.1      0.3     0.2
自动化           0.3         0.5     0.5      0.7     0.1
```

**匹配逻辑**：
- 取该专业 top-2 匹配的业务线作为内容主线和副线
- 主线的匹配度 > 0.7 时使用肯定语气（"你可以来"），0.4-0.7 使用可能性语气（"你也可以"），< 0.4 不提及
- 不同学校层次影响叙事角度：985 强调"技术创新/前沿研发"，211 强调"产业落地/工程实践"，双非强调"成长速度/晋升空间"

### 3.2 学校分层策略

| 学校层次 | 叙事重心 | 话术示例 |
|---------|---------|---------|
| **C9/顶尖985** | 前沿研发、技术主导权 | "鹏飞的电解水催化剂方向，需要你这样的基础研究人才" |
| **普通985/211** | 工程实践、产业话语权 | "从实验室到产线，你将是鹏飞氢能产业化的中坚力量" |
| **特色双非**（如化工强校） | 成长速度、弯道超车 | "化工专业去化工厂不叫对口，来鹏飞做氢能才是" |
| **普通本科** | 平台价值、职业确定性 | "鹏飞的薪酬对标一线城市，但你的生活成本是太原水平" |
| **高职院校** | 技能价值、实操不可替代 | "鹏飞的制氢工厂需要动手能力强的人，不是只会写论文的人" |

```
2. 每个业务线描述加上具体数据（规模/产量/技术参数），建立信任感
3. 避免"世界一流""国际领先"等空洞词汇——用可验证的事实说话
4. 语气根据学校层次调整：{tone_instruction}
5. 输出格式：先是一段 200 字个人化邀请语，然后分模块输出业务介绍、成长路径、传播物料

## 输出
请按照以下 JSON 格式输出：
{
  "personal_invitation": "...",    // 200字个人化邀请
  "business_intro": "...",         // 300字对口业务介绍
  "career_path": "...",            // 150字职业发展路径
  "xiaohongshu_post": "...",       // 小红书种草贴
  "bilibili_script_summary": "...",// B站视频脚本大纲
  "recommended_channels": [...]    // 推荐分发渠道
}
```

### 示例 2：计算机强校 + CS 专业

**输入**：
> 学校：浙江大学
> 专业：计算机科学与技术

**输出**：

<details>
<summary><b>📝 个人化邀请语</b></summary>

> 浙大 CS 的同学，你大概率已经刷了 300 道 LeetCode，拿了 3 个大厂实习，正在比较阿里和字节的 offer。但我想问你一个问题——你知道调度一辆氢能重卡比调度一辆外卖电动车难多少吗？外卖配送是分钟级优化，重卡物流是"制氢-储氢-运氢-加氢-运输-碳核算"六段耦合的实时优化问题。变量更多、约束更硬、容错率为零。鹏飞的零碳智慧物流平台正在用 5G+AI 做这件事——这比你在大厂做第 100 个推荐算法有意思得多。

</details>

<details>
<summary><b>💻 对口业务介绍</b></summary>

> 鹏飞的 5G 数智化业务不是传统企业的"信息化部门"——我们是在做产业级 AI 应用：
> - 工业互联网平台：把制氢工厂的上万个传感器数据实时接入，做数字孪生和预测性维护
> - 零碳物流调度引擎：在氢能重卡的续航约束、加氢站分布约束、运单时间窗约束下做全局最优调度——这是运筹学+强化学习的实战战场
> - AI 视觉质检：用 CV 做电解槽膜电极的表面缺陷检测，精度要求 μm 级
> - 碳足迹追踪系统：全链路碳排放数据的实时采集、核算和区块链存证
> 
> 这些不是"去传统企业做 IT 支撑"——你是在用 CS 能力解决真实的工业问题。

</details>

<details>
<summary><b>🎬 B站视频脚本大纲</b></summary>

### 示例 3：普通本科 + 机械工程

**输入**：
> 学校：太原科技大学
> 专业：机械设计制造及其自动化

**输出**：

<details>
<summary><b>📝 个人化邀请语</b></summary>

> 太科机械的同学，你应该知道自己学校在重型机械领域的底子有多硬。但你可能不知道的是——离你学校不到 100 公里的地方，鹏飞正在建一个氢能装备制造基地。电解槽、储氢罐、加氢机——这些不是传统重型机械，但它们对机械设计、制造工艺、材料加工的要求一点也不低。你学了四年的机械制图、机械设计、制造工艺，不用去千里之外的沿海工厂，在家门口就能做新能源装备。

</details>

<details>
<summary><b>🔧 职业成长路径</b></summary>

> **传统路径 vs 鹏飞路径**：
> 
> | 时间 | 传统机械制造企业 | 鹏飞氢能装备制造 |
> |------|:--|:--|
> | 入职 | 助理工程师，画图 | 助理工程师，直接进电解槽研发项目组 |
> | 1-3年 | 工艺工程师，盯产线 | 独立负责储氢罐某子系统的设计优化 |
> | 3-5年 | 资深工程师，带小团队 | 项目负责人，统筹氢能装备从设计到交付 |
> | 5年+ | 技术经理（天花板可见） | 氢能装备技术总监（行业人才缺口大） |
> 
> **核心理由**：传统机械制造是存量市场，论资排辈严重。氢能装备是增量市场，三年经验就是行业老人。

</details>
